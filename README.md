# 校园RPG：大学生存物语 (Campus RPG)

## 一、项目名称

**《校园RPG：大学生存物语》**（Campus RPG）

## 二、项目简介

本项目是一款基于 **C++20** 和 **Qt6** 开发的校园主题 RPG 游戏。项目不仅实现了从早期控制台文字RPG向基于 `QGraphicsView` 的**俯视角图形化地图探索**的平滑升级，更深度集成了**NPC交互、任务追踪、存档机制**等现代 RPG 的核心要素，非常适合作为 C++ 面向对象课程设计与 Qt 答辩展示的进阶范例。

玩家在校园地图中通过 `W/A/S/D` 自由探索，体验昼夜交替的校园生活。游戏内包含完善的角色属性系统、基于多态的背包系统、昼夜更替的商店与黑市系统、回合制战斗判定、拼接成整张大地图的无缝世界，以及沉浸式的任务与自动存档系统。

## 三、图形界面与核心功能说明

| 模块 | 说明 |
|------|------|
| **地图探索与碰撞** | 使用 `QGraphicsScene` 构建的 2D 俯视角地图，支持 WASD 60帧平滑移动，严格的边界碰撞检测与防穿模逻辑 |
| **无缝大世界地图** | 告别传统的场景切换与传送门，采用地图块动态拼接技术将各区域整合为一张超大无缝地图，提供沉浸式的漫游体验 |
| **NPC与任务系统** | 靠近 NPC 按键交互，支持多阶段对话与任务派发；主界面拥有动态 HUD，实时追踪当前任务进度并在接取新任务时弹出通知 |
| **存档与读档** | 拥有完整的本地 JSON 存档读取机制，并内置 5 分钟触发一次的**自动存档 (Auto-Save)** 机制，保护玩家游戏进度 |
| **角色系统与UI** | 提供图形化的角色选择界面、完整的属性面板，以及可视化的独立背包 (`BackpackWindow`) 和商店 (`ShopWindow`) |
| **昼夜与经济系统** | 后台时间系统控制昼夜交替，夜晚特定时间商店化身「黑市」，提供独特的物品池并设有强制驱逐机制 |
| **战斗与状态引擎** | 面向对象的战斗判定，支持与实体怪物交互战斗，包含瘫痪等异常状态(`StatusEffect`)检测、伤害计算与逃跑机制 |
| **JSON 数据驱动** | 瓦片地图、碰撞体、NPC 位置与属性数据均通过外部 `.json` 实时解析，实现代码逻辑与游戏数据的彻底解耦 |

## 四、核心代码模块

| 模块 | 功能说明 |
|----------|----------|
| **角色与实体 (`Character` / `Enemy`)** | 玩家与敌人的基类及属性封装（HP、攻击力、防御力等），支持面向对象的派生机制 |
| **背包系统 (`Backpack`)** | 存放/使用/丢弃物品，与基类 `Item` 的多态虚函数深度联动，实现丰富多样的道具效果 |
| **任务系统 (`Task` / `TaskWindow`)** | 管理任务的接取、条件判定与状态更新，通过 Qt 信号槽与主界面的 HUD 进行联动反馈 |
| **存档系统 (`SaveSys`)** | 将 `GameEngine` 的核心数据（角色状态、背包、任务进度、时间）序列化/反序列化为本地持久化文件 |
| **时间系统 (`GameClock`)** | 控制游戏内时间流逝，提供昼夜判断，决定场景光照渲染与黑市事件触发逻辑 |
| **交易系统 (`Shop` / `BlackMarket`)**| 独立封装白天与黑市的两套经济模型，依靠时间推进自动切换逻辑 |
| **地图系统 (`QtMapLoader`)** | 读取 `data/` 下的 JSON 文件并转化为 Qt Graphics 图元进行渲染，包含地块、碰撞层和交互层 |

## 五、创新点与技术特色

1. **彻底的图形化与UI交互**：抛弃了传统的控制台黑框，全面拥抱 Qt GUI，实现了带有动态遮罩效果的任务提示 HUD、ESC 全局菜单、以及独立的图形化背包与商店窗体。
2. **平滑按键控制与防穿模**：通过 `QTimer` 定时器（约60帧）刷新重绘玩家位置，并利用 Qt 的 Bounding Rect 计算，根除了角色走出地图边框的越界 Bug。
3. **数据驱动的设计理念**：地图地形、碰撞体积、NPC 和怪物的出生点位全面抽离至 JSON 配置文件，引擎只需一套解析器（`QtMapLoader`）即可支持无限扩充的地图规模。
4. **事件驱动的响应机制**：充分利用 Qt 的 Signal/Slot 机制，例如：接取任务后引擎发出信号，主界面捕获并自动执行 HUD 的动态弹窗提示与重新排版。
5. **底层逻辑与UI完美解耦 (MVC)**：项目底层（`GameEngine` 及其组件类）依然保持纯粹的 C++ 面向对象逻辑，不夹杂任何界面绘制代码，是标准的 MVC 架构示范。

## 六、技术亮点总结

- **语言标准**：C++20
- **GUI 框架**：Qt6 (`Core`, `Gui`, `Widgets`, `Sql`)
- **构建系统**：CMake 3.16+
- **设计模式与面向对象**：
  - **多态应用**：`Item` 基类派生出各类道具；`Enemy` 派生不同种类怪物，通过虚函数实现特定行为。
  - **单例模式**：时间管理与引擎控制等核心控制器的单例化。
- **STL 标准模板库深度使用**：大面积使用 `std::vector`, `std::set`, `std::map`, 智能指针与引用传递。
- **Qt 核心组件应用**：
  - `QGraphicsScene` / `QGraphicsView` / `QGraphicsRectItem` / `QGraphicsOpacityEffect` (高级视图与动效)
  - `QTimer` (事件循环定时刷新、自动存档轮询)
  - `QKeyEvent` (键盘事件拦截、多键同按处理)
- **跨平台资源管理**：使用 Qt `.qrc` 资源树将所有图片直接打包进可执行文件，彻底解决路径报错问题。

## 七、开发挑战说明

针对面向对象课程设计的进阶要求，本项目完成了极高难度的重构：
1. **现代化改造**：将老旧的终端字符界面成功重构为基于 **Qt QGraphics** 的现代 2D 游戏引擎标准。
2. **状态与持久化**：不仅做到了地图探索，更加入了**完整的任务状态机**和**JSON对象序列化（存档）机制**，让游戏真正具备了“可玩性”。
3. **无痛解耦**：即便进行了大幅度的图形化改造，底层业务逻辑（如背包、属性、时间、战斗）依然保持 100% 独立封装，严格贯彻了**面向对象的多态与解耦**思想。

## 九、编译与运行指南

### 环境要求
- C++20 支持的编译器 (例如 MinGW GCC 11+ / MSVC 2019+)
- CMake 3.16 及以上
- Qt6 (必须安装包含了 Core, Gui, Widgets 的组件)

### CMake 构建命令 (推荐使用 VS Code 配合 CMake Tools 插件或直接使用 Qt Creator)

命令行方式：
```bash
cd CampusRPG
mkdir build && cd build
cmake -G "MinGW Makefiles" -DCMAKE_PREFIX_PATH="C:/Qt/6.x.x/mingw_64" ..
cmake --build .
./CampusRPG.exe
```

## 十一、可写入课程设计报告的加分特色

1. **优秀的逻辑层复用 (MVC架构)**：在 `mainwindow.cpp` 引入之前，`GameEngine` 完全不知晓 Qt 的存在。这种彻底分离展现了极高的软件工程素养。
2. **事件驱动与信号槽的灵活运用**：告别死板的轮询刷新，游戏内的弹窗、任务变动追踪均采用信号槽（Signal-Slot）异步驱动，极大降低了 CPU 占用并提升了 UI 响应。
3. **安全沙箱与序列化存储**：防穿模包围盒计算保护了运行时内存的稳定；而完整的 `SaveSys` JSON 序列化实现了游戏生命周期的长久化，这是许多基础课设不具备的高级功能。
4. **全资源内存级打包**：利用 Qt RCC 机制将所有图片和 UI 资源全部编译进 EXE 文件，彻底免除了老师在验收时由于绝对/相对路径不一致导致的启动崩溃烦恼。

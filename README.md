# 校园RPG：大学生存物语 (Campus RPG)

## 一、项目名称

**《校园RPG：大学生存物语》**（Campus RPG）

## 二、项目简介

本项目是一款基于 **C++20** 和 **Qt6** 开发的校园主题 RPG 游戏。项目实现了从**早期控制台文字RPG**向**基于 QGraphicsView 的俯视角图形化地图探索**的平滑升级，非常适合作为 C++ 面向对象课程设计与 Qt 答辩展示。

玩家在校园地图中通过 `W/A/S/D` 自由探索，体验昼夜交替的校园生活。游戏内包含完善的角色属性系统、背包系统、昼夜更替的商店与黑市系统、回合制战斗判定以及无缝的地图传送功能。

## 三、图形界面与核心功能说明

| 模块 | 说明 |
|------|------|
| **地图探索** | 使用 `QGraphicsScene` 构建的 2D 俯视角地图，支持 WASD 60帧平滑移动与边界碰撞检测 |
| **场景切换(传送门)** | 走近特定坐标（如办公室、室外地图）触发传送检测，伴有黑屏过渡动画与无缝地图切换 |
| **JSON 地图加载** | 通过 `QtMapLoader` 实时解析 `.json` 格式瓦片地图数据，实现地图与代码高度解耦 |
| **昼夜系统** | 后台 `GameClock` 控制时间推进，夜晚触发独特的「黑市」商店经济系统 |
| **商店与黑市** | 白天为普通小卖部，夜晚特定时间化身黑市，提供不同的物品池并设有“防穿模强制驱逐”机制 |
| **战斗引擎** | 面向对象的战斗判定，支持攻击、使用消耗品（先手/后手设定）与逃跑机制，具备瘫痪等状态检测 |
| **背包系统** | `Backpack` 管理各类消耗品与装备，支持物品属性的多态（Polymorphism）调用 |

## 四、核心代码模块

| 模块 | 功能说明 |
|----------|----------|
| **主角系统 (`Character`)** | 玩家属性封装管理（HP、最大HP、攻击力、防御力、金币等属性计算） |
| **背包系统 (`Backpack`)** | 存放/使用/丢弃物品，与基类 `Item` 的多态虚函数深度联动 |
| **时间系统 (`GameClock`)** | 控制游戏内时间流逝，提供昼夜判断，决定场景与黑市事件触发逻辑 |
| **交易系统 (`Shop/BlackMarket`)**| 交易模块封装，白天与黑市逻辑相互独立，依靠时间流逝自动切换 |
| **战斗系统 (`GameEngine::Battle`)**| 回合制战斗状态机，包含敌人反击伤害计算、物品优先级调度等 |
| **地图系统 (`QtMapLoader`)** | 读取 `data/maps/` 下的 JSON 文件并转化为 Qt Graphics 图元进行渲染 |

## 五、创新点与技术特色

1. **图形化瓦片地图**：借助 `QGraphicsView` 与 JSON 地图资源，实现了真正意义上的 RPG 俯视角跑图体验，彻底告别纯文字。
2. **平滑按键控制与防穿模**：通过 `QTimer` 定时器（16ms，约60帧）刷新重绘玩家位置，并严格限制地图边界 `SceneRect`，根除了角色走出地图边框的 Bug。
3. **昼夜双态经济系统**：巧妙设计了白天普通商店与夜晚黑市两套经济模型，增强了校园生存的策略性。
4. **底层逻辑与UI完全解耦**：项目底层（`GameEngine` 及其组件类）独立存在，完美支持控制台（黑框版）与 Qt GUI 界面版的双向调用，是典型的 MVC 架构示范。
5. **传送过渡动画**：利用高层级 `blackMask` 遮罩层配合 `QThread::msleep`，实现了老式 RPG 经典场景切换时的黑屏淡入淡出特效。

## 六、技术亮点总结

- **语言标准**：C++20
- **GUI 框架**：Qt6 (`Core`, `Gui`, `Widgets`, `Sql`)
- **构建系统**：CMake 3.16+
- **设计模式**：
  - **多态应用**：`Item` 基类派生出各种子道具，通过虚函数实现不同疗效；`Enemy` 派生不同种类怪物。
  - **单例/管线化**：时间管理与引擎控制机制。
- **STL 标准模板库**：大面积使用 `std::vector`, `std::string`, 引用传递与指针管理。
- **Qt 核心组件应用**：
  - `QGraphicsScene` / `QGraphicsView` / `QGraphicsRectItem` (高级视图框架)
  - `QTimer` (事件循环定时刷新)
  - `QKeyEvent` (键盘事件拦截、多键同按处理)
- **跨平台资源管理**：使用 Qt `.qrc` 资源树将所有图片和 JSON 地图直接打包进 `.exe` 二进制文件，彻底解决路径报错问题。

## 七、开发挑战说明

针对面向对象课程设计的进阶要求，本组在基础要求之上完成了：
1. 将老旧的终端字符界面成功重构，接入了现代化的 **Qt QGraphics 图形框架**
2. 自主实现了 2D 坐标系下的动态移动限制和**坐标范围触发的传送门**
3. 即使图形化改造，底层业务逻辑依然保持 100% 独立封装，严格贯彻了**面向对象的多态与解耦**思想

## 九、编译与运行指南

### 环境要求
- C++20 支持的编译器 (例如 MinGW GCC 11+ / MSVC 2019+)
- CMake 3.16 及以上
- Qt6 (必须安装包含了 Core, Gui, Widgets 的组件)

### CMake 构建命令 (推荐使用 VS Code 配合 CMake Tools 插件)

命令行方式：
```bash
cd CampusRPG
mkdir build && cd build
cmake -G "MinGW Makefiles" -DCMAKE_PREFIX_PATH="C:/Qt/6.x.x/mingw_64" ..
cmake --build .
./CampusRPG.exe
```

> **对照提示**：目前 `main.cpp` 已全面接入 Qt GUI。如果您想体验纯控制台命令行版逻辑，可将 `main.cpp` 中部的 `#if 0` 宏开关开启进行验证。

## 十一、可写入课程设计报告的加分特色

1. **优秀的逻辑层复用 (MVC架构)**：在 `mainwindow.cpp` 引入之前，`GameEngine` 完全不知晓 Qt 的存在。这种彻底分离展现了极高的软件工程素养。
2. **现代化 C++ 特性**：代码级限定 C++20 标准，放弃老旧冗余写法。
3. **动态寻路与安全沙箱**：利用 Qt 包围盒 API 计算（Bounding Rect）彻底解决了游戏开发中最常见的“角色走出地图”导致越界崩溃的恶性 Bug。
4. **全资源内存级打包**：利用 Qt RCC 机制将外部资源全部编译进内存，彻底免除了因绝对路径、相对路径引起的跨平台运行崩溃烦恼。
